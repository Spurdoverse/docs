Everything on this page is available in every script without any import.
These are the building blocks every script starts with: scheduling work,
reacting to game events, and reading per-frame state.

## Timers

Standard browser-style timer API. IDs returned by `setTimeout` and
`setInterval` are independent — pass the right ID to the right clear function.

### setTimeout

<br />

```ts
setTimeout(callback: () => void, ms: number): number
```

Calls `callback` once after `ms` milliseconds. Returns a timer ID.

```ts
setTimeout(() => {
    console.log("3 seconds have passed");
}, 3000);
```

---

### setInterval

<br />

```ts
setInterval(callback: () => void, ms: number): number
```

Calls `callback` repeatedly every `ms` milliseconds. Returns a timer ID.

```ts
const id = setInterval(() => {
    console.log("tick");
}, 1000);
```

---

### clearTimeout / clearInterval

<br />

```ts
clearTimeout(id: number): void
clearInterval(id: number): void
```

Cancels a pending timer by its ID. Safe to call with an invalid or
already-fired ID.

```ts
const id = setInterval(() => {
    console.log("tick");
}, 500);

// Stop after 5 seconds
setTimeout(() => clearInterval(id), 5000);
```

<Warning>
  Always cancel intervals you no longer need. Intervals keep firing even if
  the entities or objects they reference have become invalid, and they
  accumulate across script reloads until explicitly cleared.
</Warning>

## console

Standard console output. All methods accept any number of arguments
and format them automatically.

```ts
console.log(...args: any[]): void      // general output
console.warn(...args: any[]): void     // yellow warning
console.error(...args: any[]): void    // red error
console.assert(cond: any, ...args: any[]): void  // logs if cond is falsy
```

```ts
console.log("Player HP:", player.m_iHealth);
console.warn("Entity not found for index", idx);
console.error("Critical failure:", err);
console.assert(player.isValid, "Player should be valid");

// Multiple values are space-joined
console.log("pos:", pos.x, pos.y, pos.z);
```

## Events

### on

<br />

```ts
on(event: string, callback: (...args: any[]) => void): number
```

Registers a callback for a named event. Returns a numeric **ID** that
can be passed to [`off`](#off) to unregister.

```ts
const id = on("tick", () => {
    // called every tick (128 Hz)
});
```

---

### once

<br />

```ts
once(event: string, callback: (...args: any[]) => void): number
```

Like `on`, but the callback fires **once** and is automatically removed.

```ts
once("tick", () => {
    console.log("only the first tick");
});
```

---

### off

<br />

```ts
off(id: number): boolean
```

Removes a callback by its ID. Returns `true` if found and removed,
`false` if already removed or invalid.

```ts
const id = on("tick", () => { /* ... */ });

off(id);   // true  — removed
off(id);   // false — already gone
```

<Tip>
  Always store the ID returned by `on()` if you plan to clean up later.
  Use `once()` when you only need a single invocation — it handles
  cleanup for you.
</Tip>

### Built-in events

#### `"tick"`

Fires every game tick (128 Hz). The primary place for per-frame logic —
entity reads, render calls, state machines.

```ts
import Entities from "@native/entities";
import Render   from "@native/render";

on("tick", () => {
    const local = Entities.getLocalPlayer();
    if (!local) return;

    const { success, screen } = Math.worldToScreen(local.m_pGameSceneNode.m_vecAbsOrigin);
    if (success) {
        Render.text(screen, `HP: ${local.m_iHealth}`, Color.white());
    }
});
```

---

#### `"OnMouseMove"`

```ts
on("OnMouseMove", (absX: number, absY: number, deltaX: number, deltaY: number) => void)
```

| Param | Type | |
|:------|:-----|:--|
| `absX` | `number` | Absolute cursor X position in screen pixels |
| `absY` | `number` | Absolute cursor Y position in screen pixels |
| `deltaX` | `number` | Change in X since the last event |
| `deltaY` | `number` | Change in Y since the last event |

```ts
let cursorPos = { x: 0, y: 0 };

on("OnMouseMove", (absX, absY) => {
    cursorPos = { x: absX, y: absY };
});
```

---

#### `"OnMouseClick"`

```ts
on("OnMouseClick", (button: number, isDown: boolean) => void)
```

| Param | Type | |
|:------|:-----|:--|
| `button` | `number` | `0` = left, `1` = right, `2` = middle, `3` = X1, `4` = X2 |
| `isDown` | `boolean` | `true` on press, `false` on release |

```ts
on("OnMouseClick", (button, isDown) => {
    if (button === 0 && isDown) {
        console.log("Left mouse button pressed");
    }
});
```

---

#### `"OnMouseScroll"`

```ts
on("OnMouseScroll", (direction: number) => void)
```

| Param | Type | |
|:------|:-----|:--|
| `direction` | `number` | `1` = scroll up, `-1` = scroll down |

```ts
on("OnMouseScroll", (dir) => {
    console.log(dir > 0 ? "Scrolled up" : "Scrolled down");
});
```

---

#### `"OnKeyDown"` / `"OnKeyUp"`

```ts
on("OnKeyDown", (vkCode: number) => void)
on("OnKeyUp",   (vkCode: number) => void)
```

| Param | Type | |
|:------|:-----|:--|
| `vkCode` | `number` | Windows Virtual Key code |

```ts
const VK_INSERT = 0x2D;

on("OnKeyDown", (vk) => {
    if (vk === VK_INSERT) {
        console.log("Insert pressed");
    }
});
```

<Tip>
  Virtual Key codes are listed in the
  [Microsoft documentation](https://learn.microsoft.com/en-us/windows/win32/inputdev/virtual-key-codes).
  Common ones: `0x01` LMB · `0x02` RMB · `0x04` MMB · `0x10` Shift ·
  `0x11` Ctrl · `0x12` Alt · `0x2D` Insert · `0x2E` Delete.
</Tip>

### onUnload

Script cleanup is handled by exporting a function named `onUnload`.
It is called once when the script is about to be unloaded or reloaded.

```ts
export function onUnload() {
    console.log("script unloaded");
}
```

<CodeGroup>
```ts Cleanup intervals
const intervalId = setInterval(() => { /* ... */ }, 100);

export function onUnload() {
    clearInterval(intervalId);
    console.log("Cleaned up.");
}
```

```ts Cleanup event listeners
const tickId = on("tick", () => { /* ... */ });
const keyId  = on("OnKeyDown", () => { /* ... */ });

export function onUnload() {
    off(tickId);
    off(keyId);
}
```
</CodeGroup>

<Warning>
  `onUnload` must be a **named export** at the module level. It will not
  fire if declared as a local variable or inside a nested function.
</Warning>

## script

The `script` object exposes metadata about the currently running script.
All fields are **readonly**.

```ts
declare var script: {
    name: string;
    version: string;
    author: string;
};
```

| Field | Type | Description |
|:------|:-----|:------------|
| `name` | `string` | Script display name |
| `version` | `string` | Version string |
| `author` | `string` | Author name |

```ts
console.log(`${script.name} v${script.version} by ${script.author}`);
```

## Recipes

### Feature toggle with a keybind

```ts
let enabled = false;
const VK_INSERT = 0x2D;

on("OnKeyDown", (vk) => {
    if (vk === VK_INSERT) {
        enabled = !enabled;
        console.log("Feature:", enabled ? "ON" : "OFF");
    }
});

on("tick", () => {
    if (!enabled) return;
    // ... render / logic
});
```

### Throttle expensive work with setInterval

```ts
import Entities from "@native/entities";
import Render   from "@native/render";

let cachedEnemies = [];

setInterval(() => {
    cachedEnemies = Entities.find("C_CSPlayerPawn")
        .filter(p => p.m_iHealth > 0)
        .toArray();
}, 100);

on("tick", () => {
    for (const enemy of cachedEnemies) {
        const { success, screen } = Math.worldToScreen(enemy.m_pGameSceneNode.m_vecAbsOrigin);
        if (success) Render.circleFilled(screen, 4, Color.red());
    }
});
```

### Frame-time independent animation

```ts
import Game   from "@native/game";
import Render from "@native/render";

let alpha = 0;
let direction = 1;

on("tick", () => {
    alpha += direction * 120 * Game.frameTime;

    if (alpha >= 255) { alpha = 255; direction = -1; }
    if (alpha <= 0)   { alpha = 0;   direction =  1; }

    const { x: w, y: h } = Render.getScreenSize();
    Render.text(
        { x: w / 2, y: h / 2 },
        "Hello World",
        new Color(255, 255, 255, alpha)
    );
});
```
